    动态规划¶

    特征：¶

    当前最优解的前束所有的最优解都已经解决

    1. 最优子结构：就是说，问题的子问题也是最优的（贪心也具有）
    2. 子结构可以多次在后面的状态应用：F(3)，F(6)=F(3)+w(3),F(9)=F(3)+w(6),F是当前的最优解，这个是明显特征。
    3. 前面的状态可以改变

    价值一定，用最少的个数¶

    当前子问题的解将由之前已经解决的子问题某一个或几个的解推出，不一定是前一个子问题的解，两个相邻问的解肯那个没有关系。

    首先我们思考一个问题，现在有1,3,5三种硬币，如何用最少的硬币凑够i元(i<11)？为什么要这么问呢？两个原因：

    1.当我们遇到一个大问题时，总是习惯把问题的 规模变小，这样便于分析讨论。

    2.这个规模变小后的问题和原来的问题是同质的，除了规模变小，其它的都是一样的，本质上它还是同一个问题(规模变小后的问题其实是原问题的子问题)。

    动态规划是一步步规模变大的问题。

    现在我要拼7元钱

    解决方案是：

    • 当i=0,d(i)=0;

    • i=2之前只有一种硬币：

    • i=1时，d(1)=d(1-1)+1

    • i=2 ，d(2)=d(2-1)+1;

    • i=3-5有两种硬币可以选择:

    • i=3时,①d(3-1)+1;

    • ②d(3-3)+1;

    • d（3）=min(①，②)

    • i=4,d(4)=min(d(4-1)+1,d(4-3)+1);

    • i=5,d(5)=min(d(5-1)+1,d(5-3)+1,d(5-5)+1);

    • i=6,d(6)=min(d(6-1)+1,d(6-3)+1,d(6-5)+1);

    • i=7,d(7)=min(d(7-1)+1,d(7-3)+1,d(7-5)+1)//就三种硬币可以选择就是

    实际上是一种递归问题。

    个数（体积等）一定，装最大的价值¶

    经典的背包问题：一定得容量，装足够多的的价值¶

    后续其他背包问题背包问题：

    实际上就是：

    容量m,最佳价值w

    多个：

    m=1,w(1)

    m=2,w(2)=max(w(1)+weight(1),weight(2))

    m=3,w(3)=max(weight(3),w(2)+weight(1),w(1)+weight(2))

    m=4,w(4)=max(weight(4),w(3)+weight(1),w(2)+weight(2),w(1)+weight(3))

    每件物品只有一个：

    表格法：

    某一件物品放与不放的影响

    F(i,j)表示在体积为j的别背包中放前i件物品的最优解：

    每一次，都在放该第i件物品与不放中挣扎比较

    ①如果放：要预留出i物品的体积m(i)，然后加上i的价值weight。F(i-1,j-m(i))+weight(i);

    ②如果不放：价值跟前i-1一样 ：F(i-1,j)

    F(i,j)取两种情况的最大值： $$ F(i,j)=max(F(i-1,j-m(i))+weight(i),F(i-1,j)) $$

    问：贪心能解决这个问题？不能 ，因为空间的限制，虽然我们每次可以取价值量/体积最大的东西，但是由于空间上的束缚，你不知道，这个东西是否能存的上，且存入之后，另外的多余空间是否能存上其他物品，不能就造成浪费，与贪心的本质矛盾。

    可以改写成一维数组：

    for(i=1;i<=m;i++){  //硬币种类  
    for (j=n;j>=m[i];j--){  
        F[j]=min(F[j],F[j-m[i]]+weight(i)) ;
    }

    到这里，会看出跟硬币问题的别，是j--为什么要倒着那：

    首先我们要看出跟哪道题的区别：这道题是有限个物体，每件物品只能拿一次。

    我们假设正序，且有以下可能：

    i=3时，F(3)的最优解，F(6)=F(3)+w(3)是F（6）的最优解，F（9）=F（6）+w(3)是F(9)的最优解，也就是说F（9）的最优解里，有多个3组成的，但实际3只有一个

    现在倒序：

    i=3,j=9时，F（9)的最优解还是F（6）+w(3)的话，F（6)一定不包含3的存在，因为此时的F(6)还没有更新i=3的情况，还是i=2的最优解，同理F(6)最多有一个2。且当j=6时才决定是否要3

    一定，求种类个数¶

    小A点菜：求种类个数¶

    思路：在吃与不吃中抉择：

    F(i,j)是前i件物品，花刚好j元的种类个数

    若吃：F(i-1,j-w(i)) *1

    若不吃：F(i-1,j)

    当然，还要吃得起才可以吃

    if (j>w(i)) 

F(i,j)=F(i-1,j-w(i))+F(i-1,j)

else F(i,j)=F(i-1,j)

    初始化F(i,0)=1 因为所有钱买一道菜也是一种方案。

    滑雪问题¶

    #include<iostream>
#include<cstdio>
#include<cstring>
#include<algorithm>
using namespace std;
const int max_size=110;
int R,C;
int dir[4][2]={{-1,0},{0,1},{1,0},{0,-1}};//四个方向
int h[max_size][max_size],dp[max_size][max_size];
int inMap(int x,int y){//判断是否在地图内
    if(x>=0&&x<=R-1&&y>=0&&y<=C-1) return 1;
    return 0;
}
int max2(int a,int b,int c,int d){//寻找最大值
    return max(max(a,b),max(c,d));
}
int dfs(int i,int j){
    int nx,ny,down=0,up=0,left=0,right=0;
    if(dp[i][j]) return dp[i][j];
    nx=i+dir[0][0]; ny=j+dir[0][1];//第一个方向
    if(inMap(nx,ny)){
        if(h[i][j]>h[nx][ny]) up=dfs(nx,ny);
    }
    nx=i+dir[1][0]; ny=j+dir[1][1];//第2个方向
    if(inMap(nx,ny)){
        if(h[i][j]>h[nx][ny]) right=dfs(nx,ny);
    }
    nx=i+dir[2][0]; ny=j+dir[2][1];//第3个方向
    if(inMap(nx,ny)){
        if(h[i][j]>h[nx][ny]) down=dfs(nx,ny);
    }
    nx=i+dir[3][0]; ny=j+dir[3][1];//第4个方向
    if(inMap(nx,ny)){
        if(h[i][j]>h[nx][ny]) left=dfs(nx,ny);
    }
    dp[i][j]=max2(up,down,left,right)+1;//判找四个方向哪个最长！是最长！
    return dp[i][j];
}
int main(){
    scanf("%d%d",&R,&C);
    memset(h,0,sizeof(h));
    memset(dp,0,sizeof(dp));
    for(int i=0;i<R;i++){
        for(int j=0;j<C;j++){
            scanf("%d",&h[i][j]);
        }
    }
    int ans=-1;
    for(int i=0;i<R;i++){
        for(int j=0;j<C;j++){
            ans=max(ans,dfs(i,j));
        }
    }
    printf("%d\n",ans);
}

    高等动态规划¶

    这里说高级，其实也不是很难的意思。前面我们学的都是线性动态规划等一般可以用二维数组可以解决的。

    现在，我们见一见其他的不可以或者难以用动态规划解决的。

    有以下几种：

    状态DP，区间DP，多维DP

    石子归并¶
