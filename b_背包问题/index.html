    动态规划——背包问题

    01背包问题：¶

    01背包问题的特点：一定的容量，装足够多的物品

    容量m,最佳价值w

    多个：

    m=1,w(1)

    m=2,w(2)=max(w(1)+weight(1),weight(2))

    m=3,w(3)=max(weight(3),w(2)+weight(1),w(1)+weight(2))

    m=4,w(4)=max(weight(4),w(3)+weight(1),w(2)+weight(2),w(1)+weight(3))

    每件物品只有一个：

    表格法：

    某一件物品放与不放的影响

    F(i,j)表示在体积为j的别背包中放前i件物品的最优解：

    每一次，都在放该第i件物品与不放中挣扎比较

    ①如果放：要预留出i物品的体积m(i)，然后加上i的价值weight。F(i-1,j-m(i))+weight(i);

    ②如果不放：价值跟前i-1一样 ：F(i-1,j)

    F(i,j)取两种情况的最大值： $$ F(i,j)=max(F(i-1,j-m(i))+weight(i),F(i-1,j)) $$ 问：贪心能解决这个问题？不能 ，因为空间的限制，虽然我们每次可以取价值量/体积最大的东西，但是由于空间上的束缚，你不知道，这个东西是否能存的上，且存入之后，另外的多余空间是否能存上其他物品，不能就造成浪费，与贪心的本质矛盾。

    完全背包问题：¶

    每件物体都是无限个

    #include<bits\stdc++.h>
using namespace std;
//完全背包问题 ：每件物品的有无限个 

int main(){
    int n,m;//n是物品种类数，m是可以容纳的体积
    int a[i],w[i];
    int i,j;
    cin>>n>>m;
    //二维数组方法 
    for(i=1;i<=n;i++){
        for(j=1;j<=m;j++){
            int t=0;
            while(1){
                if(j>t*a[i]){
                    F[i][j]=max(F[i][j],F[i-1][j-t*a[i]]+t*w[i]);//t从0开始，选择最优解
                    //在只有i-1种类的物品里，扣去花费t件物品i的体积，加上t件i物品的价值，比较找出max 
                }
                else break;
                t++; 
            }
        }
        int ans=0;
        //比较寻找max值 
    } 
    //一维数组方法 ：前面我们已经验证了，如果j从0开始选择的话，是基于前面的物品无限件来选的 
     //比如：F[3]的最优解是选择一个种类3的物品，F[6]的最优解是F[6]=F[6-3]+w[3],
     //那么F[6]的最优解是用了两个种类3物品

        for(j=1;j<=m;j++){

    多重背包问题：¶

    就是 每种物品不是无限件，比如种类1物品有：10件，可以把10 拆成:1 2 4 3,

    前几位是二进制表示法，后面最后一位数是10-1-2-4；

    拆成的这几个数，可以表示1-10任意一个数，也就表示可以选择种类1物品1-10任意件组成最优解
